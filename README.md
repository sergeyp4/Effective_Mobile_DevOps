## Запуск проекта ##
1. Клонировать репозиторий: `git clone https://github.com/sergeyp4/Effective_Mobile_DevOps.git`
2. В директории репозитория выполнить: `docker compose up`

## Проверка ##
Выполнить `curl http://localhost`. Ожидаемый вывод: **"Hello from Effective Mobile!"**

## Схема прохождения трафика ##
Трафик, пришедший на порт **80** docker-хоста, направляется на порт **8080** контейнера **nginx-reverse-proxy**, а nginx в свою очередь проксирует трафик на порт **8080** контейнера **backend**.
Использование порта **8080** вместо **80** на nginx требуется из соображений безопасности, чтобы nginx мог работать под непривилегированным пользователем (для прослушивания портов ниже 1024 требуются привилегии root).

## Особенности реализации ##
- Использовались минималистичные базовые образы для экономии дискового пространства и трафика (при скачивании образов из реестра)
- Для повышения безопасности приложения в контейнерах работают от непривилегированного пользователя. Для этого пришлось использовать особенный образ Nginx и сменить порт Nginx c 80 на 8080.
- Для Health Сheck используется специальное минималистичное приложение, т.к. curl в образах отсутствует. Это же приложение можно было бы использовать и для Nginx, но тогда пришлось бы использовать собственный образ, на основе базового. Учитывая, что в Docker Compose Health Сheck не влияет на работу контейнера (в Docker Swarm "нездоровые" контейнеры могли бы перезапускать автоматически), такой подход мог бы стать лишним усложнением.
- В целях безопасности у контейнеров забираются все CAPABILITIES и запрещается получение новых привилегий.
